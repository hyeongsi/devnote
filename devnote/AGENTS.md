# 작업 규칙

Codex는 매 작업마다 `docs/` 아래의 모든 문서를 읽지 않는다.

현재 변경과 직접 관련된 문서만 골라서 참고한다.

# 참고 문서

- Git 커밋 및 PR 규칙:
  - `docs/git-rules.md`

# 인코딩 안전 규칙

- 프로젝트의 텍스트 파일은 UTF-8 기준으로 다룬다.
- 한글이 포함된 파일을 수정할 때는 기존 인코딩과 줄바꿈 방식(CRLF/LF)을 유지한다.
- 작업에 꼭 필요하지 않다면 파일 전체를 다시 쓰지 않는다.
- 터미널 출력이 깨져 보일 경우, 그 깨진 문자열을 그대로 소스 파일에 옮기지 않는다.
- 한글 문자열과 무관한 작은 범위만 최소한으로 수정하는 패치를 우선한다.
